# 練習題

## 基礎題

- 目標：實作一個可對資料清單進行排序與搜尋的函式，並確保結果一致。
- 限制：請勿直接使用 Python 內建的 `list.index` 來實作搜尋邏輯，排序得維持穩定性。
- 輸入：清單（包含字典或數字），以及欲搜尋的目標值
- 輸出：排序後的清單與搜尋到的索引位置
- 評分要點：排序結果是否正確、搜尋邏輯是否正確、測試是否涵蓋邊界情況

## 延伸題

- 情境變更：資料中包含重複值與混合大小寫字串，請讓排序與搜尋能正確處理這些情況。
- 你需要做的調整：為排序與搜尋增加 key 函數支援，並寫測試驗證不同鍵值下結果是否一致。
- 交付物：程式、測試、簡短說明

## 反思題

- 用自己的話說明：為什麼穩定排序在某些資料整理流程中比單純快一點更重要？
- 若資料來源改成每天更新一批新紀錄，你會如何改寫搜尋流程？
