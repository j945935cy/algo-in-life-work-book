# 第五章：圖與路徑：從流程網路找到最短可行路線

::: advisory
本章目標
:::

- 理解節點、邊與權重如何描述真實流程
- 用 Python 建立可重現的最短路徑求解流程
- 把通勤、簽核與任務依賴轉成可比較的路徑成本

::: advisory
情境與限制
:::

- 情境：你要在多條候選路線中選出成本最低的一條，例如通勤路徑、簽核流轉或跨部門協作流程
- 輸入：節點關係、邊權重、起點與終點
- 輸出：最短總成本與完整路徑
- 限制：圖可能不是完全連通，且必須能清楚指出找不到路徑的情況

::: advisory
可重現規則
:::

- 本章程式碼位置： `examples/ch05/`
- 本章核心模組： `examples/ch05/routing.py`
- 測試： `pytest -q tests/test_ch05_graphs_and_routing.py`
- 典型用途：通勤規劃、工單流轉、任務依賴分析

## 本章內容概要

本章會把常見的「從 A 到 B 要怎麼走比較省」問題，轉成帶權重的圖。接著用 Dijkstra 演算法計算最短路徑，並把最終路徑重新還原成可閱讀的節點序列。

## 實作範例

請參考 `examples/ch05/routing.py`。

示範流程包含：

- 用字典定義圖的鄰接關係與權重
- 計算從起點到終點的最短成本
- 還原完整經過路徑
- 處理無路可達的情況
