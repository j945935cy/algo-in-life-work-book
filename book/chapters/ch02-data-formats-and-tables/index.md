## 本章內容概要

很多人第一次碰資料問題，卡住的地方其實不是分析，而是資料本身就已經亂掉了。

同一欄有的像數字、有的像文字；日期有人寫得完整、有人只填一半；看起來像空白的地方，實際上又不是真的空值。這些問題如果不先整理，後面做再多分析都很容易歪掉。

本章會從最常見的表格資料出發，示範如何建立一套簡單但穩定的資料轉換流程。重點不是背格式名稱，而是知道什麼時候該把資料轉成什麼樣子，才比較好保存、重用和檢查。

## 為什麼資料格式會影響理解

同一份資料，放在不同格式裡，能保留的資訊不一定一樣。

例如有些格式比較適合人直接打開閱讀，有些格式比較適合機器快速載入，有些格式則比較適合拿來保存處理後的乾淨版本。

如果你沒有先分清楚「原始資料」「處理中資料」「分析用快照」這幾種角色，後面很容易把檔案越存越亂。

## 本章想幫你建立的習慣

這一章最想建立的是三個習慣：

1. 先定義欄位應該長什麼樣子。
2. 遇到缺值和格式錯誤時，用一致規則處理。
3. 把處理後結果存成適合後續使用的格式。

## 實作範例

請參考 `examples/ch02/` 中的 `data_transform.py`。本章會建立一個可執行的資料轉換腳本，並示範如何在 `data/` 內保存原始資料、處理後資料與快照。

示範流程包含：

- 從 CSV 與 JSON 載入原始資料
- 依照欄位型別定義進行型別正規化
- 輸出 Parquet 檔案與 NumPy 快照檔案
