# 第一章：用演算法思維整理日常工作

::: advisory
本章目標
:::

- 建立演算法思維：把日常工作轉成輸入、輸出、限制、成本
- 透過可執行範例與測試，養成可重現的工作流程

::: advisory
情境與限制

- 輸出：排序後的行事曆與工作建議

## 本章內容概要

很多人會以為演算法是一種很遠的東西，但其實它常常只是「把事情整理清楚的方法」。

例如今天工作很多，你會先想：哪些事情比較急？哪些事情可以晚點做？哪些事情其實只要十分鐘，但拖著不做反而一直卡在心裡？這些判斷，本身就很接近演算法思維。

本章的重點不是馬上追求很厲害的技巧，而是先練習把一個模糊問題，整理成可以討論的形式。

## 先不要急著寫程式

在開始寫程式之前，更重要的是把問題的輪廓講清楚。至少要先知道：

1. 你手上有什麼資訊。
2. 你最後想得到什麼結果。
3. 過程裡有哪些限制不能忽略。

如果這三件事講不清楚，程式通常也只是在放大混亂。

## 演算法思維不是只有「快」

很多入門書一提到演算法，就很快進到速度比較。但在日常工作裡，很多時候你真正需要的，反而是：

- 決策邏輯能不能說明白
- 同一批資料重做一次時，結果會不會差太多
- 別人能不能看懂你是怎麼排出這個順序的

所以這本書會反覆強調「可重現」這件事。因為能重現，才容易溝通，也才容易修正。

## 實作範例

請參考 `examples/ch01/time_log.py`。

示範流程包含：

- 先把待辦事項拆成可比較的欄位
- 根據優先度、截止日和預估時間做排序
- 產出比較容易執行的工作順序
- 用測試固定基本判斷規則
